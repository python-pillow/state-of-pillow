Packaging fork

Pillow is a "friendly" fork of the Python Imaging Library. The goal is to see if any improvement can be made to the packaging situation by opening up development to the public.

—Alex Clark, Pillow 1.0 README

First the first two years, Pillow was a "packaging fork"
https://mail.python.org/pipermail/image-sig/2012-December/007120.html
We added filesystem paths to setup.py, made other small changes, and attempted to upstream fixes to PIL.

Unfriendly fork?

https://mail.python.org/pipermail/image-sig/2012-December/007120.html
Then a guy named Brian Crowell (fluggo) came along and added support for Python 3. I sent a mail Image-SIG about our inability to continue tracking changes upstream.
Python 3 support came in two PRs.
This felt really good.
Pillow 2.0 with support for Python 3 was released on March 15, 2013.

Python 3 support

The "little fork that could"
Pillow >= 2.0.0 supports Python versions: 2.6, 2.7, 3.2, 3.3
Pillow < 2.0.0 supports Python versions: 2.4, 2.5, 2.6, 2.7
This is when things started to get really exciting.
